# 实验与论文设计

本文档从《详细设计.md》中拆分出来，只描述对比实验、评价指标、论文实现路径和实验产物。

### 1. 基线方法与对比实验设计

这个模块非常重要，因为初版也要求导出“有/无评论区对比实验数据”给论文组。你们要把这个变成正式实验。

#### 1.1 方法对比

| 方法 | 输入 | 技术 | 作用 |
| --- | --- | --- | --- |
| Baseline A | 主贴 | 关键词匹配 | 模拟传统舆情系统 |
| Baseline B | 主贴 | 情感词典/简单情感模型 | 判断负面倾向 |
| Baseline C | 主贴 + 评论 | 大模型直接判断 | 证明裸 LLM 的不稳定 |
| Ours-1 | 主贴 + 评论 + 语义向量 | 语义表示 + 评论链演化 | 证明语义集中和时间顺序有用 |
| Ours-2 | 主贴 + 评论 + 图谱 + RAG | 语义表示 + 图谱评分 + 证据检索 | 最终方法 |

#### 1.2 评价指标

```text
风险等级准确率 Accuracy
高风险召回率 High-risk Recall
关键评论命中率 Key Comment Hit Rate
阶段识别准确率 Stage Accuracy
处置建议人工评分 Human Rating
JSON有效率 Schema Valid Rate
```

#### 1.3 实验输出

```text
outputs/experiment_results/
├── baseline_keyword.csv
├── baseline_sentiment.csv
├── llm_direct.csv
├── ours_semantic_chain.csv
├── ours_full.csv
└── ablation_summary.csv
```

#### 1.4 消融实验

```text
完整系统
- 去掉评论区
- 去掉时间顺序
- 去掉图谱中心性
- 去掉RAG证据
- 去掉大模型解释
```

你们论文里就写：

> 实验表明，仅基于主贴的关键词或情感方法容易漏判反讽、情绪共振和组织化倾向；引入语义向量表示、评论链结构、时间演化和关键评论排序后，系统能够更准确地定位风险升级节点。

### 2. 论文实现路径

论文不要写产品商业模式。初版也已经说明论文线要围绕“基于对话链的校园舆情语义分析框架”，并做“有/无评论区效果对比”的消融实验。你们最终论文可以升级成：

**《面向校园墙场景的评论链驱动舆情风险演化识别方法研究》**

论文方法章节对应代码模块：

```text
3.1 数据采集与脱敏          -> anonymize_service.py
3.2 评论链建模              -> comment_graph.py
3.3 语义表示与特征构建      -> embedding_service.py / semantic_features.py
3.4 风险信号识别            -> risk_scoring.py
3.5 情绪共振指数            -> risk_scoring.py
3.6 关键评论排序            -> key_comment_ranker.py
3.7 大模型结构化研判        -> llm_service.py
3.8 RAG证据化处置           -> rag_service.py
```

实验章节对应：

```text
4.1 数据集构建
4.2 标注规范
4.3 对比方法
4.4 消融实验
4.5 结果分析
4.6 案例分析
```
