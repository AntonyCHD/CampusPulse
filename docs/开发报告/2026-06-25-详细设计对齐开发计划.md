# 2026-06-25 详细设计对齐开发计划

## 有效约束

- 当前前端技术栈以 `frontend-vue/` 为准，继续使用 Vue 3 + TypeScript + Element Plus；不恢复早期 Streamlit 目录。
- `docs/问题确认.md` 已确认 RAG 先用硬编码规则库，后续有知识库数据后再替换为检索式实现。
- `docs/详细设计.md` 的 M7/M8/M9 是本批次重点：证据化处置、结构化报告、前端展示闭环。
- 现有 MVP 代码优先复用，不新建平行系统。

## 本批次目标

1. 后端新增硬编码证据检索服务，返回 `Evidence[]`，让报告页有真实证据链而不是空模板。
2. 报告 API 返回结构接近 `FinalReport`：包含证据、关键评论解释、处置建议、人工复核标记和生成时间。
3. 事件列表 API 增加总览统计：今日事件数、高风险事件数、风险等级分布、事件类型分布、最近风险趋势。
4. Vue 事件总览页展示统计图表，报告页展示证据链与关键评论解释。
5. 清理仍会误导开发者的 Streamlit 依赖/部署描述，保持文档和当前实现一致。
6. 新增后端单元测试和 API 集成测试；运行前端构建验证。

## 文件计划

- 新增 `backend/app/services/rag_service.py`：硬编码规则证据库与关键词匹配排序。
- 修改 `backend/app/api/routes_report.py`：接入证据服务，补全报告字段和 Markdown 导出。
- 修改 `backend/app/api/routes_events.py`：返回统计摘要，保持 `items` 兼容现有前端。
- 修改 `frontend-vue/src/api/client.ts`：补全事件、统计、报告类型。
- 修改 `frontend-vue/src/views/EventList.vue`：增加统计卡片、风险分布图、类型分布图、趋势图。
- 修改 `frontend-vue/src/views/ReportExport.vue`：展示证据链、关键评论解释、人工复核状态。
- 修改 `pyproject.toml`、`docs/开发规范与部署.md`、`docs/详细设计.md`：同步 Vue 技术栈，移除当前依赖中的 Streamlit 遗留项。
- 新增 `backend/tests/test_rag_service.py`、`backend/tests/test_events_api.py`、`backend/tests/test_report_api.py`。

## 测试计划

1. 先写失败测试：
   - `pytest backend/tests/test_rag_service.py -q`
   - `pytest backend/tests/test_events_api.py backend/tests/test_report_api.py -q`
2. 实现后运行：
   - `pytest backend/tests -q`
   - `npm run build`，工作目录 `frontend-vue/`
3. 若服务可启动，再运行脚本级集成检查：
   - `python scripts/test_api.py`

## 不做事项

- 不恢复 Streamlit。
- 不接入实时 LLM API。
- 不引入真实向量数据库或外部知识库。
- 不改动原始数据采集流程。
