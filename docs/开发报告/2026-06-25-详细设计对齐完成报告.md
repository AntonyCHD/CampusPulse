## 1. 后端新增与改动

### 1.1 RAG 证据服务
- 文件：`backend/app/services/rag_service.py`
- 实现：`RagService` — 基于硬编码规则库的确定性证据检索
  - 5 条处置证据规则（宿舍诉求、后勤回应、学生诉求、线下聚集、传闻澄清）
  - 关键词匹配 + 优先级排序，返回 `Evidence[]`
  - 单例 `get_rag_service()` 供 API 层直接使用
- 对应 `docs/详细设计.md` M7 的“基础版返回手工模板”

### 1.2 事件列表 API 增加总览摘要
- 文件：`backend/app/api/routes_events.py`
- 变更：`GET /api/events/` 返回体增加 `summary` 字段
  - `total_events`、`today_events`、`high_risk_events`
  - `risk_distribution` → 风险等级柱状图数据
  - `event_type_distribution` → 事件类型饼图数据
  - `risk_trend` → 风险趋势折线数据
- 对应 `docs/详细设计.md` 9.1 的事件总览指标

### 1.3 报告 API 增加证据与关键评论解释
- 文件：`backend/app/api/routes_report.py`、`backend/app/schemas/report.py`
- 变更：
  - 报告响应新增 `evidence`、`key_comment_explanations`
  - 新增 `human_review_required` 和 `generated_at`
  - Markdown 导出包含证据链和关键评论解释
- 对应 `docs/详细设计.md` 8.11 的 `FinalReport` Schema

### 1.4 测试
- 测试文件：
  - `backend/tests/test_rag_service.py` — RAG 证据检索，2 条通过
  - `backend/tests/test_events_api.py` — 事件总览 summary，1 条通过
  - `backend/tests/test_report_api.py` — 报告 evidence 字段，1 条通过
