# 答辩展示与交付清单

本文档从《详细设计.md》中拆分出来，只描述答辩演示路径、技术亮点、交付清单和最终实施建议。

### 1. 答辩演示路径

现场演示别乱点，固定剧本：

```text
第一步：打开事件总览
  展示系统不是单案例脚本，而是有事件列表和风险排序。

第二步：选择“宿舍限电争议”
  展示主贴和评论区。

第三步：点击“运行分析”
  展示风险阶段从“信息求证”到“情绪共振”再到“组织化行动”。

第四步：展示关键评论
  指出哪条评论导致风险升级。

第五步：切到“对比实验”
  传统关键词/情感法只识别到负面情绪，但语义评论链法识别出组织化行动。

第六步：展示证据化处置
  系统给出温和回应话术、行动建议、责任部门和避免话术。

第七步：导出报告
  证明系统有实际应用闭环。
```

### 2. 最终技术亮点提炼

答辩时可以讲 5 个创新点：

#### 创新点一：从“关键词匹配”升级为“语义表示”

系统用中文语义向量理解评论之间的相似诉求、反讽表达和同类情绪，而不是只看词表命中。

#### 创新点二：从“帖子级判断”升级为“评论链演化判断”

传统方法看主贴，我们看主贴、评论、回复、时间和互动。

#### 创新点三：提出校园舆情情绪共振指数

不是简单负面情感，而是综合情绪比例、语义集中、互动放大和时间密度。

#### 创新点四：关键评论定位

系统能指出风险升级的触发评论，而不是只给一个分数。

#### 创新点五：证据化处置

处置建议来自校规、通知、历史案例和模板检索，不是大模型凭空发挥。

### 4. 最终建议版本

你们现在不要从“大而全”开始写代码。最稳路线是：

```text
第 1 个闭环：
脱敏案例 -> 语义向量缓存 -> 风险评分 -> 结构化报告 -> 前端展示

第 2 个闭环：
关键词/情感基线对比 -> 关键评论定位 -> 对比实验页

第 3 个闭环：
语义评论图谱 -> 情绪共振指数 -> 风险演化路径

第 4 个闭环：
RAG证据检索 -> 处置建议 -> 报告导出
```

最小可演示核心是：

> 选择一个校园墙事件，系统展示评论区如何把事件从“个体吐槽”推向“情绪共振”甚至“组织化行动”，并指出关键评论、给出证据化处置建议。

这就是你们项目的灵魂：
**不是压住声音，而是听懂声音；不是监控学生，而是让问题在变成风暴前被看见。**
